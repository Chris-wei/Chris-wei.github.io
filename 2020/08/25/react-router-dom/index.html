手动实现一个 react-router-dom

发表于 2020-08-25 | 分类于 React

引言

我们日常中使用 react 开发项目，那么一定会跟 react-rouer 打交道，但是由于 react 路由的设计和用法，使得很多刚使用 react 的同学在接触到react-router的时候就会很蛋疼，不知道其匹配的机制和原理。

追本溯源一直是我们学习和提升过程中一个很重要的过程，本文就以 HashRouter 为例，结合最新的 react-hooks 语法， 一步步的实现一个属于自己的 react-router，看一看 react-router 到底应该怎么实现！

初始化项目

我们需要做一些前期的准备工作，初始化项目这一步是必须的，我们通过

npx create-react-app my-react-router

脚手架工具初始化一个项目，为了方便测试，将里面没用的文件删除掉，保留一个比较干净的 workspace。

Redirect 组件

有时候我们希望用户访问到某一路由时重定向到其他路由，例如我们如果输入不合法的路由，匹配不成功的时候我们希望重定向到我们的首页 /home 上去，就需要用到 Redirect 组件。

在我们的 react-router-dom 文件夹下新建 Redirect.js 文件并在 index.js 导出。

Redirect 组件与 Link 类似，只不过它不做热河渲染，加载完毕就直接跳转。

这会，你随便敲一个不存在路径，就会帮你重定向到 home 页面~

但是，你再点击 category、profile 路由的时候也会闪一下后重定向到 home 页面。

这是因为，我们并没有做处理，虽然匹配到了 category 和 profile 路由，但是也并不妨碍继续往下执行，就又到了 Redirect 组件，应该说是我们无论如何匹配都会重定向到 home 页面，这显然是不合理的，需要通过 Swtich 组件进行包裹。

Swtich 组件

Swtich 组件要实现的功能就是当我们匹配到了路径之后，就不需要继续匹配，解决上述重定向的问题。

我们在 react-router-dom 文件夹下新建 Swtich.js 文件并在 index.js 导出。

实现思路就是，循环遍历 Switch 的子组件，判断当前路径是否与子组件的路由匹配，匹配成功后返回即可。
